# Decisions Documentation

Repositorio de decisiones técnicas y funcionales de KeyGo.

## Contenido

- [adr](adr): decisiones aceptadas y estables.
- [rfcs](rfcs): propuestas, planes y gaps de diseño.
- [rfc-closure-process.md](rfc-closure-process.md): cómo absorber RFCs al canon.

## Convenciones

- Crear un ADR cuando una decisión ya esté activa en código o documentación canónica.
- Mantener un RFC cuando el diseño aún está en discusión, implementación parcial o discovery.
- Si hay tensión entre documentos, registrar el conflicto; no reescribir la historia.
