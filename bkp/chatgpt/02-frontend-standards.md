# Estándares Frontend — KeyGo UI

## Objetivo
Definir reglas claras para mantener consistencia, calidad y mantenibilidad.

---

## Estructura de carpetas
- Usar organización por feature
- Evitar mezclar dominios

---

## Código
- Tipado estricto (TypeScript)
- Evitar any
- Funciones puras cuando sea posible

---

## API
- Nunca llamar HTTP desde componentes
- Usar capa api/

---

## Hooks
- Un hook = una responsabilidad
- Evitar hooks gigantes

---

## Formularios
- Usar React Hook Form
- Validaciones con schema (Zod/Yup)

---

## UI
- Componentes reutilizables
- Evitar duplicación

---

## Naming
- consistente y descriptivo

---

## Performance
- evitar renders innecesarios
- usar memo cuando aplique
