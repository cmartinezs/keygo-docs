# Security Policy

## Supported Versions

This is a documentation template repository. We do not release versioned software.
All content on the `main` branch is considered current and supported.

## Reporting a Vulnerability

If you discover a security vulnerability in this repository — for example,
sensitive data accidentally committed, malicious links in templates, or
infrastructure concerns related to the project — please report it responsibly.

### How to Report

**Please do not open public issues for security vulnerabilities.**

Instead, please contact the maintainers privately through GitHub security advisory or direct contact via the repository owner.

Include the following details:
- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix or mitigation (if any)

### Response Process

1. **Acknowledgment**: We will acknowledge receipt within 48 hours.
2. **Investigation**: We will investigate and validate the report within 5 business days.
3. **Resolution**: If confirmed, we will work on a fix and coordinate disclosure.
4. **Disclosure**: Once resolved, we will publish a summary and credit the reporter (unless anonymity is requested).

## Scope

This security policy covers:
- Content within this repository
- Linked resources managed by the maintainers
- Examples and templates that could lead to insecure practices if flawed

It does **not** cover:
- Security of products documented using this template
- Third-party tools or services referenced in the documentation

## Best Practices for Users

When using this template for your own projects:
- Never commit secrets, API keys, or credentials into documentation.
- Review AI-generated content for accidental data leakage before committing.
- Keep your documentation repository access controls aligned with your project policies.

Thank you for helping keep this community safe.
