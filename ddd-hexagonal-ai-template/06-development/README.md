## Overview

En esta fase documentamos:

1. **Architecture** — Hexagonal / Limpia / por capas, mapa de módulos a Bounded Contexts
2. **API Reference** — Contratos REST/GraphQL
3. **Coding Standards** — Convenciones, patrones, anti-patterns
4. **ADRs** — Decisiones técnicas con alternativas consideradas

**Entregable**: Guía completa de cómo construir. Developers saben cómo empezar.

**Duración típica**: 3-4 semanas (paralela con Testing)

**Audiencia**: Developers, Tech Leads, DevOps

---

## Notas

- **THIS PHASE IS TECH-SPECIFIC** — Aquí mencionas tu stack (Java, Python, Node, etc.)
- Mapea Bounded Contexts del Design a paquetes/módulos
- Documenta patrones DDD tácticos (Aggregates, Repositories, Value Objects, etc.)
- Incluye ejemplos reales de código
