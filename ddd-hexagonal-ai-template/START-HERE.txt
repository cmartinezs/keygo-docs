PHASES OVERVIEW
================================================================================

Phase 0: PLANNING            Setup framework & conventions
Phase 1: DISCOVERY           Understand the business problem
Phase 2: REQUIREMENTS        Define what the system must do
Phase 3: DESIGN              Design flows and UI/UX
Phase 4: DATA MODEL          Define entities and relationships
Phase 5: PLANNING            Create roadmap and sprints
Phase 6: DEVELOPMENT         Build architecture and code
Phase 7: TESTING             Test strategy and validation
Phase 8: DEPLOYMENT          CI/CD and release process
Phase 9: OPERATIONS          Runbooks and incident response
Phase 10: MONITORING         Metrics and alerts
Phase 11: FEEDBACK           Retrospectives and learning

Each phase builds on the previous one. Complete each phase before moving to
the next.

================================================================================

IMPORTANT PRINCIPLES
================================================================================

1. PHASE DISCIPLINE
   Phases 1-2 (Discovery & Requirements): WHAT, not HOW
   - Include: Business needs, constraints, actors
   - Exclude: Technology names, frameworks, code patterns

2. SINGLE SOURCE OF TRUTH
   All documentation lives here. Other repos link to or submodule this repo.

3. TRACEABILITY
   Link requirements through design → development → testing
   Use IDs: FR-001, SF-001, TC-001 for consistent references

4. AI-ASSISTED, HUMAN-LED
   AI can generate templates and drafts. Humans validate and customize.

5. CONTINUOUS DOCUMENTATION
   Update as decisions are made, don't batch at the end.

TIPS & BEST PRACTICES
================================================================================

1. Start with Discovery
   Don't jump to implementation. Understand the problem first.

2. Use Templates
   Each section has EXAMPLE subsections. Copy them and customize.

3. Link Everything
   Every requirement should trace back to discovery and forward to design.

4. Get Sign-Off
   Don't move to next phase without stakeholder approval.

5. Use AI Assistance
   Let AI generate initial drafts, you refine them.

6. Keep It Live
   Update documentation as you learn, not just at phase end.

7. Preserve History
   Old decisions go to 01-templates/data-input/ for archival, not deleted.

FREQUENTLY ASKED QUESTIONS
================================================================================

Q: How long does this take?
A: 8-12 weeks for a medium project (5-10 people). Phases can overlap.

Q: Can we skip phases?
A: Not recommended. Each builds on previous ones.

Q: Can we use AI to write all the docs?
A: No. AI generates drafts (50%), humans refine (50%).

Q: What if we're already building?
A: Retrofit documentation from what you've decided so far.

Q: How do we keep docs in sync with code?
A: Automation for APIs/schemas, code reviews for consistency.

More FAQ: 00-guides-and-instructions/FAQ.md
