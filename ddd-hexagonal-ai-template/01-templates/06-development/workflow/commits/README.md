# Commit Conventions

## Description

How to write commit messages.

## Format

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

## Types

| Type | Description |
|------|-------------|
| feat | New feature |
| fix | Bug fix |
| docs | Documentation |
| style | Formatting |
| refactor | Code refactoring |
| test | Tests |
| chore | Maintenance |
| perf | Performance |

## Examples

```
feat(auth): add login

fix(user): validate email format

docs: update README
```
