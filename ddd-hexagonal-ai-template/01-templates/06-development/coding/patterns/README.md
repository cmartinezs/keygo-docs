# Design Patterns

## Description

Common reusable solutions for recurring problems.

## Categories

| Category | Focus |
|----------|-------|
| **Creational** | Object creation |
| **Structural** | Object composition |
| **Behavioral** | Object communication |

---

## Creational Patterns

| Pattern | Description | When to Use |
|---------|-------------|-------------|
| **Factory** | Create objects without specifying exact class | Object creation with variations |
| **Abstract Factory** | Create families of related objects | Multiple object types |
| **Builder** | Construct complex objects step by step | Objects with many parameters |
| **Singleton** | One instance globally | Shared resources |
| **Prototype** | Clone existing objects | Replicate state |

## Structural Patterns

| Pattern | Description | When to Use |
|---------|-------------|-------------|
| **Adapter** | Convert interface to another | Legacy integration |
| **Bridge** | Decouple abstraction from implementation | Multiple implementations |
| **Composite** | Tree structures | Hierarchies |
| **Decorator** | Add behavior dynamically | Extensions |
| **Facade** | Simplified interface | Complex subsystem |
| **Proxy** | Placeholder for another object | Lazy loading, access control |

## Behavioral Patterns

| Pattern | Description | When to Use |
|---------|-------------|-------------|
| **Observer** | Notify changes to multiple | Event systems |
| **Strategy** | Interchangeable algorithms | Multiple algorithms |
| **Command** | Encapsulate request as object | Undo/redo, queues |
| **State** | Object changes behavior with state | State machines |
| **Chain of Responsibility** | Pass request along chain | Middleware |
| **Template Method** | Define algorithm skeleton | Frameworks |
